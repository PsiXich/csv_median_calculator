## 📖 Описание

**CSV Median Calculator** — профессиональное приложение для анализа биржевых данных. Читает файлы с историей торгов, вычисляет медиану цен по мере обработки данных и сохраняет результаты.

### Ключевая особенность

Медиана рассчитывается **инкрементально** — каждое новое значение учитывает все предыдущие:

```
Прочитали цену 100 → медиана 100
Добавили цену 102  → медиана 101
Добавили цену 99   → медиана 100
```

Используется **P² алгоритм** из Boost.Accumulators для эффективного расчёта без хранения всех значений в памяти.

## 📦 Установка

### Подробная инструкция по сборке

#### Windows (MinGW/MSYS2)

```bash
# 1. Открыть MSYS2 MinGW64 терминал
cd csv_median_calculator

# 2. Создать директорию для сборки
mkdir build && cd build

# 3. Генерация проекта
cmake .. -DENABLE_STRICT_WARNINGS=OFF

# 4. Компиляция (используем все ядра)
cmake --build . -j$(nproc)
```

#### Linux

```bash
# 1. Установить зависимости (Ubuntu/Debian)
sudo apt-get update
sudo apt-get install build-essential cmake git

# 2. Сборка
mkdir build && cd build
cmake ..
cmake --build . -j$(nproc)
```

#### macOS

```bash
# 1. Установить инструменты (если нет)
xcode-select --install
brew install cmake

# 2. Сборка
mkdir build && cd build
cmake ..
cmake --build . -j$(sysctl -n hw.ncpu)
```

### Проверка установки

```bash
cd build
./csv_median_calculator --help
```

Если видите справку — всё установлено правильно! ✅

## ⚙️ Конфигурация

### Формат файла config.toml

```toml
[main]
    # Путь к директории с входными CSV-файлами (обязательно)
    input = './examples/input'
    
    # Путь к директории для выходных файлов (опционально)
    # По умолчанию: ./output
    output = './examples/output'
    
    # Список масок имен файлов для фильтрации (опционально)
    # Пустой список = все CSV файлы
    filename_mask = ['level', 'trade']
```

### Параметры

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `input` | string | Да | Директория с входными CSV файлами |
| `output` | string | Нет | Директория для результатов (по умолчанию `./output`) |
| `filename_mask` | array | Нет | Фильтр файлов по имени (пусто = все файлы) |

- Header-only библиотека для парсинга TOML
- Поддержка TOML 1.0
- Безопасная обработка ошибок

#### spdlog (1.13.0)

- Быстрое логирование
- Форматирование через fmt
- Настраиваемые паттерны вывода

- Автоматическое скачивание зависимостей
- Кроссплатформенная сборка
- Опциональные строгие предупреждения компилятора

## 📊 Примеры

### Формат входных данных

#### level.csv

```csv
receive_ts;exchange_ts;price;quantity;side;rebuild
1716810808593627;1716810808574000;68480.00000000;10.10900000;bid;1
1716810808593627;1716810808574000;68479.90000000;0.00400000;bid;0
```

**Поля:**
- `receive_ts` — временная метка получения (микросекунды)
- `exchange_ts` — временная метка биржи (микросекунды)
- `price` — цена
- `quantity` — объём
- `side` — сторона (`bid` или `ask`)
- `rebuild` — флаг перестроения (0 или 1)

#### trade.csv

```csv
receive_ts;exchange_ts;price;quantity;side
1716810808663260;1716810808661000;68480.10000000;0.01100000;bid
1716810809314641;1716810809312000;68480.00000000;0.00200000;ask
```

**Поля:**
- `receive_ts` — временная метка получения (микросекунды)
- `exchange_ts` — временная метка биржи (микросекунды)
- `price` — цена сделки
- `quantity` — объём сделки
- `side` — сторона (`bid` или `ask`)

### Формат выходных данных

#### median_result.csv

```csv
receive_ts;price_median
1716810808663260;68480.10000000
1716810809314641;68480.05000000
1716810809719209;68480.10000000
```

**Важно:** Запись производится **только при изменении медианы**.
